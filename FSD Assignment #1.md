#### Welcome to your first step at ERIC Robotics!  
This assignment is designed to assess your ability to transform a UI/UX design into a fully functional, self-hosted dashboard. The design you’ll be implementing is our **`Insight.IO`** dashboard (below). Your goal is to faithfully recreate the layout, components, and interactions shown in the demo.

(![Insight dashboard](https://gist.github.com/user-attachments/assets/993770ca-b24d-47d6-94c3-8c394490a99a)

---

## **Assignment Requirements**

### **1. Setup**
- **Clone or download** this gist.
- Study the provided GIF carefully — note the layout, animations, and interactions.

### **2. Implementation**
You can use:
- Any modern frontend framework (**React**, **Vue**, **Angular**, etc.)
- Or plain **HTML/CSS/JavaScript** if you prefer.

Your dashboard must:
- Match the design and interactivity of the demo.
- Be **responsive** and **user-friendly**.
- Include:
  - **Camera View** – Use either a provided video file or a live feed.
  - **3D Map View** – Use a `.pcd` file or integrate with ROS via tools like `roslibjs` to play bag files.

### **3. Self-Hosting**
- Set up a local development server (Node.js, Python, or any static server).
- Ensure your implementation can run locally without internet dependencies (except for any framework libraries you choose).

### **4. Documentation**
Your submission must include:
- **Clear setup instructions** so we can run your project locally.
- A **short explanation and video/image** of your approach, tools used, and any design/architecture decisions.

---

## **Bonus Points**
- Modular, maintainable code structure.
- ROS integration instead of static files.
- Clear commit history showing progressive work.

---

## **Submission Guidelines**
- Push your code to a **public GitHub repository**.
- Ensure your documentation is complete.
- Deadline: 3 days from acceptance of assignment.
---

### **Evaluation Criteria**
We’ll be looking at:
- Accuracy of implementation against the provided design.
- Code quality, readability, and maintainability.
- Responsiveness and UX polish.
- Clarity of your documentation.
- Sample image/video of your dashboard.
